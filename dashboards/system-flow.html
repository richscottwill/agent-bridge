What this system is

The system is a pipeline orchestrator, not a chatbot. Richard's inputs (Slack, meetings, email, tasks) get captured continuously. Twice a day — AM at 6:30 and EOD at ~18:00 — backend hooks fan out to parallel subagents that read, summarize, and reconcile. The output lands on this dashboard, in Slack as a daily brief, and as writes into Asana (next-action fields, enrichment, completion).

Two other pipelines run on their own cadences. The WBR callout pipeline (analyst → writer → reviewer) produces market callouts every Tuesday from MotherDuck ps.v_weekly. The wiki team (editor → researcher → writer → critic → librarian) publishes long-form articles to SharePoint's Artifacts/ library. Both read the same facts but emit different artifacts.

The body system (Overview · Agent Activity tabs) is meta-telemetry on the orchestrator itself — hook reliability, workflow success rates, autoresearch loop health, experiment keep-rate. If this page shows what the system produced today, Body System shows whether the machinery is healthy. See: Body System →

What breaks when Failure modes, ranked by frequency
Loop docs auth expires
Every 7–10 days the Loop MCP loses KMSI auth. AM Backend logs all 5 pages as 401 and skips. Fix: mwinit -f.
Nested subagent blocker (karpathy Phase 6)
From 2026-04-22: z14.registerSubAgentExecution is not a function. Blocks all blind-eval experiments. EOD runs, Phase 6 silently skipped. Autoresearch loop stalled 8+ days.
MOTHERDUCK_TOKEN missing in hook env
When hooks shell out to Python (hard-thing-refresh.py, refresh-body-system.py), the token isn't in env — silent null-state writes. Fix shipped 2026-04-30: scripts now auto-resolve from mcp.json, fail-loud if missing.
Hedy MCP unavailable in subagents
Meeting transcripts work from orchestrator but not from parallel subagents — roster doesn't include mcp_hedy_*. Workaround: run Hedy ingestion in EOD only, not AM parallel.
Asana My Tasks 502s
GetUserTasks hits SSE errors intermittently. AM pulls 5/6 projects and flags the miss rather than failing the whole run. No user impact beyond a warning in the brief.
Slack + MotherDuck — resilient
Slack MCP hasn't missed a run in 30+ days. MotherDuck writes across all 10 schemas are reliable once attached. These are the workhorses — if they break, almost nothing else matters.
