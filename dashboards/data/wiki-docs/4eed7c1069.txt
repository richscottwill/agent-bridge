- **Decision-forcing function:** the vocabulary update is contingent on blackboard success, which is itself at 0% adoption. Both are staged but neither is moving.

### 2. Frontmatter quoting inconsistency

Some articles wrap values in double quotes (`status: "DRAFT"`), others don't (`status: DRAFT`). Both parse, but inconsistency is a sign nobody is enforcing a schema linter. Files drafted 2026-04-17 use quoted form; earlier files use unquoted.

**Recommendation:** pick one (unquoted — YAML-idiomatic), add a pre-commit lint. Not urgent — parsers handle both.

### 3. Audience vocabulary drift

Values seen in corpus:
- `amazon-internal` (majority)
- `team` (2026-04-17 drafts)
- `personal`
- `leadership`
- `agent` (new — for agent-facing docs like meeting-ingestion-pipeline)
- `org`
- `self`

Seven values with overlap. `team` vs `amazon-internal` vs `org` are effectively the same audience. `self` and `personal` are the same.

**Recommendation:** consolidate to 4 values (`team`, `leadership`, `personal`, `agent`). Cheap normalization pass, no blackboard dependency.
