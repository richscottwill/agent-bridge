# AI Automation Command Center — Paid Search

Richard Williams · Amazon Business, Paid Acquisition · Started March 2026
Last measured: 2026-03-19 · Next measurement: 2026-04-19

---

## Automation Status

🟢 Done by agents (no human input needed): 11%
🟡 Human + agent (augmented): 34%
🔴 Human only (not yet automated): 55%

These percentages are based on a weighted inventory of recurring weekly workflows that have been observed so far (see full breakdown below). Weights reflect approximate time spent per week. Updated monthly.

Important caveat: This inventory is incomplete by design. It reflects what has surfaced through email, calendar, task tracking, and direct conversation — not the full scope of work. There are almost certainly workflows, ad hoc tasks, and judgment-heavy work that haven't been captured yet. As we work together and more work becomes visible, these percentages will recalibrate. Expect the "human only" percentage to increase before it decreases — not because automation regressed, but because the denominator got more accurate.

## Full Workflow Inventory

Every recurring workflow I do, categorized by automation level. This is the source of truth for the percentages above.

### 🟢 DONE BY AGENTS (human reviews output, doesn't create it)

Daily planning + prioritization
- What: Scan email/calendar/Asana, prioritize tasks, generate daily brief
- How: Morning Routine hook + Asana Sync hook + Run the Loop hook (3 hooks, 1 click each)
- Before: ~45 min/day · After: ~15 min/day (review only)
- Weekly savings: ~2.5 hrs

Context maintenance across sessions
- What: Maintain source-of-truth files (current projects, people, meetings, actions)
- How: Autoresearch Loop — scans email/calendar, updates context files, cascades changes
- Before: ~30 min every 2 weeks · After: ~2 min (bootstrap loads automatically)
- Weekly savings: ~0.2 hrs (amortized)

Research artifact compression
- What: Compress large bodies of context into small, high-signal files
- How: Autoresearch Loop — 8 experiments, all kept. Artifacts: compressed context, predicted Q&A, relationship graph, decision log, metric snapshot, action tracker, meeting briefs, master brief
- Before: Agent re-read source material every session (~27 tool calls) · After: 0 tool calls
- Impact: ~27 tool calls saved per session, faster + more accurate responses

### 🟡 HUMAN + AGENT (agent drafts, human reviews/edits/sends)

Email drafting (routine replies)
- What: Draft replies to stakeholder emails, status updates, data requests
- How: Writing Style System (7 steering files) + Draft Unread Replies hook + task body drafts
- Before: ~12 min each, ~8/week · After: ~3 min each (review + edit)
- Weekly savings: ~1.2 hrs

Task creation + management
- What: Create tasks with status, action steps, draft messages, priority
- How: AI generates task bodies in To-Do (4-list workflow: Sweep/Core/Engine Room/Admin)
- Before: ~15 min each, ~5/week · After: ~3 min each
- Weekly savings: ~1.0 hr

WBR callout prep
- What: Prepare traffic/conversion callouts for weekly business review
- How: Agent pre-drafts callouts from available data, I review and adjust
- Before: ~40 min/week · After: ~15 min/week
- Weekly savings: ~0.4 hrs

Meeting prep + follow-up
- What: Prep notes for recurring syncs (AU, MX, Adi, Brandon, Google, Adobe)
- How: Meeting Briefs artifact provides position + talking points. Agent drafts follow-up messages.
- Before: ~20 min per meeting, ~6 meetings/week · After: ~10 min per meeting
- Weekly savings: ~1.0 hr

Calendar blocking for focus time
- What: Create calendar events with task context for deep work blocks
- How: Task Calendar Block hook — creates events with full task body
- Before: ~5 min per block · After: ~1 min (one click)
- Weekly savings: ~0.2 hrs

Strategic document drafting
- What: Write frameworks, test designs, POVs (e.g., OP1 Testing Framework, AEO POV)
- How: Agent produces first draft from context + research artifacts, I restructure and refine
- Before: ~3-4 hrs per doc · After: ~1.5-2 hrs per doc
- Frequency: ~1/week (target) · Savings per occurrence: ~1.5-2 hrs

Patterns that could scale without each person building their own setup.

### Ready now (proven, low effort)

- Writing style steering files — AI drafts in a consistent voice per context. Each person creates their guide once.
- Session bootstrap files — Persistent context across chat sessions. Template + 30 min setup.

## Systems Built

### Daily Operations System (3 hooks, working together)
- Asana Sync — Scans Outlook for Asana notifications → extracts task IDs → syncs to To-Do
- Run the Loop — Refreshes context files from email/calendar, runs experiments, cascades changes
- Morning Routine — Reads all context → prioritizes → triages → daily brief → emails it
- These run in sequence: Asana Sync feeds To-Do → Loop feeds context → Morning Routine synthesizes both.

### Communication System (2 hooks + 7 steering files)
- Draft Unread Replies — Generates draft replies for unread emails
- Block Email Send — Safety check before any email send
- Writing Style: core + email / WBR / MBR / docs / Amazon / Slack variants

### Knowledge System (autoresearch loop + 8 artifacts)
- Compressed Context (15:1) · Predicted Q&A (8:1) · Relationship Graph (10:1) · Decision Log (12:1) · Metric Snapshot (8:1) · Action Tracker (6:1) · Meeting Briefs (9:1) · Master Brief (20:1)
- ~27 tool calls saved per session

### Accountability System (trainer + tracker + bootstrap)
- RW Trainer — detects low-leverage patterns, pushes toward strategic work
- Task Management — 4-list workflow (Sweep/Core/Engine Room/Admin) + Backlog
- Session Bootstrap — persistent memory across 14-day chat expiry
- Task Calendar Block — creates focus events with full context

## How I Measure

Most organizations measure AI impact by asking employees how much time they think they save. That's not measurement.

Four metrics, updated monthly:

1. Time per task (before vs. after) — Measured in minutes. Includes AI processing + human review time.
2. Redirection quality — Where did saved time go? High-leverage or low-leverage?
3. System reliability — % of AI outputs needing only minor edits. Target: >80%.
4. Compounding indicator — Is the system improving over time? Tracked via autoresearch metrics.
