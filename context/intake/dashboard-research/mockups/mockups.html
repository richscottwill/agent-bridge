Ship order — recommended for kiro-server

These are the T1 items from the full report, ordered so each commit is a standalone improvement that doesn't depend on later work. Every mockup above has a spec block detailing the file + selector + data source.

  1. M1 — Sticky header + trust bar. Absorbs region + metric filters. Biggest top-of-fold win; enables everything else. 1 commit · ~2h
  2. M2 — Headline + exception banner. Replaces opening paragraph with one-line WBR-style headline and conditional red-lane exception. 1 commit · ~1.5h
  3. M3 — KPI cards with sparklines + bullet for vs-OP2. Drops global metric filter; row shows all metrics as side-by-side cards. Also ships the shared renderSparkline + renderBullet helpers used by M4, M6, M8. 1 commit · ~2h
  4. M4 — Bullet chart component (shared). Ships with M3 but usable anywhere. MPE distance-to-target tab is the second consumer. Piggybacks M3
  5. M5 — Reliability diagram + signed-error bars. Replaces 4-card calibration scorecard. Keep the 4 numbers as a compact footer strip. 1 commit · ~2.5h
  6. M6 — Small multiples across 12 markets. New section between scorecard and main chart. Click-to-select reuses M1 market selector. 1 commit · ~2h
  7. M7 — Amazon 6-12 chart. Swaps the default trend panel for two side-by-side (6wk weekly / 12mo monthly) with box-score footer. Lives in canon-chart.js. 1 commit · ~3h
  8. M8 — Prior-week thread as sparkline strip. Replaces text pager. Reuses renderSparkline. 1 commit · ~1h
  9. M9 — Fan chart on MPE. Projection engine side; replaces text range. 1 commit · ~2h
  10. M10 — Waterfall variance decomposition. Replaces the WoW table. 1 commit · ~1.5h

Total ~18h of implementation across 10 commits. All 10 items are independently shippable — kiro-server can pick any order that matches current context.

How to verify each commit

For every mockup MN, the verification probe is identical:

1. Start dashboard server: serve.py from agent-bridge/dashboards/ (must be this root, not workspace root)
2. Open http://localhost:8080/performance/weekly-review.html?market=US&week=W17
3. Compare visually against mockups/screenshots/MN-*.png
4. Run through the 12 trust-bar markets to confirm state transitions
5. Test sticky behavior by scrolling 2,000px down; header stays pinned
6. Keyboard: J/K for week nav, / to focus market, ? for shortcuts (when M15-M17 land)
