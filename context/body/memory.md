## Usage Guide — Meeting Prep & Common Failures

**Meeting Prep:** Series files live in `~/shared/wiki/meetings/` — one file per recurring series.

| Subfolder | Covers |
|-----------|--------|
| `meetings/stakeholder/` | AU, MX, Google, Adobe, MCS/Polaris, OCI |
| `meetings/team/` | Weekly sync, deep dive, promo OHs, PAM, activation |
| `meetings/manager/brandon-sync.md` | Brandon 1:1 |
| `meetings/peer/` | Adi, Yun, Andrew |
| `meetings/adhoc/adhoc-log.md` | One-off meetings |

Each series file contains metadata, context, latest session summary, running themes, and open items. Use the relationship graph for tone/draft style; use series files for meeting-specific dynamics, decisions, and action items.

**Common Failures:**
1. **Pronoun error on Brandon.** Brandon uses she/her pronouns — zero tolerance for error. Before any draft mentioning Brandon: check relationship graph → confirm she/her → write. Worked example: "She mentioned in our 1:1 that the Testing Approach needs revision" — never "he" or "they."
2. **Register mismatch.** Match tone to relationship depth. Established contacts (e.g., Lena): lead with data — "AU CPA is $140 — here's the plan." Newer contacts (e.g., Lorena): warm opener first — "Hi Lorena, I wanted to share the latest on AU CPA performance." Always read the relationship graph entry before drafting.
3. **Carlos ≠ Lorena on MX.** MX PS stakeholder changed on 3/17. Carlos moved to CPS. Lorena is now the PS stakeholder. Never conflate them — they hold different roles in different orgs.
4. **Stale interaction dates.** If last interaction exceeds 14 days, dynamics may have shifted. Check the Relationship Staleness Index before drafting. Worked example: Yun-Kang (last interaction 3/19, 17d stale) → flag "Low risk but stale" → open with a brief check-in before making the ask.
