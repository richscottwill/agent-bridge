# Richard Williams — Personal Operating System

AI-augmented work system. Text files that let any AI platform understand how Richard thinks, works, and operates.

## Quick Start

1. Read `context/body/body.md` — maps the whole system
2. Read `context/body/spine.md` — bootstrap sequence and key IDs
3. Read `DIRECTORY-MAP.md` — where every file type belongs

## Top-Level Structure

| Directory | Purpose |
|-----------|---------|
| context/ | Live context: organs, active state, meetings, wiki, archive |
| artifacts/ | Published work product, frameworks, testing, weekly ships |
| data/ | DuckDB databases, raw/processed data, exports, per-market data |
| tools/ | Automation tools, team tools, autonomous workflows |
| uploads/ | Human drop zone (sheets, changelogs, docs, scratch) |
| research/ | Active research files |
| portable-body/ | Cold-start bootstrap kit |
| audit-reports/ | System audit outputs |
| .kiro/ | Agents, hooks, specs, steering, settings |

See `DIRECTORY-MAP.md` for full detail on every subdirectory.
