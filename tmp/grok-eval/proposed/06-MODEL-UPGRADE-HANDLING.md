# Model Upgrade Handling Protocol (Opus and future versions)

When Richard announces a model upgrade (e.g., Opus 4.7 → newer):

1. Immediately review `soul.md` + this file + `STEINBERGER-AGENTIC-ENGINEERING-UPGRADES.md`.
2. Identify any steering, routing, or principle language that should be strengthened or simplified for the new model's strengths/weaknesses.
3. Propose a short diff of changes (maximum 5 bullet updates).
4. After Richard approves, apply the changes and run a test session on a real task (e.g., latest test readout or projection).
5. Update this file with lessons learned from the upgrade.

This keeps the system current without manual heavy lifting.
