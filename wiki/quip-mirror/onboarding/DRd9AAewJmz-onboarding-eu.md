# On-boarding to EU
## **Ways of working: **
* **First steps: **
 * Start get familiar with EU Account structure, multilingual and multi country structure.
 * Once new alias gets assigned, get permissions across the board. Google Ads, Bing Ads, Coupa.
 * Start to take over accounts
 * Transfer POs: “Requested by” on the POs.
  * Richard will handle the Q1 Microsoft POs, Andrew starts to take over Microsoft POs after
* **Daily tasks: **
 * Monitor EU budget: pacing report
 * Optimization check
* **Weekly tasks: **
 * Weekly reporting for Paid Search meeting
 * WBR callouts to prepare every Monday by 5pm PST for EU (Meeting is weekly Tuesday at 2pm)
  * Katy/Stacey will message in the “ab-paid-search-global” channel once the dashboard is ready
 * Check WBR report to see if there’s any huge data discrepancy against our dashboard
 * Check Auction Insight and Search terms quality
* **Monthly tasks: **
 * MBR/QBR: Brandon (prepare quick notes)
 * MSP: Email coming to us (invoice email), and we make sure it matches
 * R&O: York Chen- Yun is taking care of R&O for EU5 SSR
* ** Quarterly or 2 times/year:**
 * OP2 approval, sync with Brandon, then PO. (Takes about a month)
 * Create PO, longer process, want before Invoice expires in January.
  * Google POs- Yun, Microsoft POs- Richard/Andrew
 * Update PO in platform invoices/budgets.

## **EU Campaigns: **
The structure is the same on EU5 level, but of course in terms of keywords we made sure to accommodate the top searched terms for each country. For example "pizza restaurant supplies" works well in IT, in Germany do not have the translation but it has been localized to match the market so it will be localized to a proxy which is "Snack bar supplies" [i.e. imbiss großhandel]. Most important info about the structure below**.**

#### **1) Overview: **
* Unbranded/Branded/AMZ/Other
* Campaigns by Vertical/Product/Feature
* Audience based campaigns GA- GA2  (google audience), RLSA
* Match Type split on Ad group  level
* Negative cross Adgroups/  Generic Negative lists
#### **2) Campaign structure: **
* Brand (core, GA etc contains  keywords like "amazon business")
* AMZ (replicating the entire  structure adding "amazon")
* Vertical (Office, Education,  Medical, Food, Hospitality, Beauty etc contains keywords like "office  supplies")
* Products By vertical (Product  Electronic will contain terms like Computer, Office Product will contain  terms like "pens" etc)
* Shops (Retail Dedicated  campaigns contains keywords like "supplies for retailers")
* Material (contains raw  materials terms contains keywords like "buy raw materials in  bull")
* Feature (Tax, Invoicing etc  contains keywords like "pay by invoice")
#### **3) Ad group structure (based on the modifiers) **
* Generic
* Purchasing
* Suppliers
* Distributors
* Wholesales
* Procurement
* Bulk

#### **2) EU tracking (ref tags):**
****For PS, please follow the following naming convention for using reftags: (1) prefix + (2) country code + (3) suffix + (4) 3 digits number
* (1) Channel prefix for PS is  "b2b_reg_search" (Branded= b2b_reg_search_branded)
* (2) Country Code are the 2  initials of the Amazon Marketplace (DE, UK, FR, IT, ES,JP)
* (3) There is one unique  reftag per keyword
* (4) Keep it a 3-digits so it  is easy to isolate - this is to differentiate templates and versions for  same segments within same marketplace
* _Examples_:
 * Vertical Office:  b2b_reg_search_IT_VER_Off_ + numeric value
 * Vertical Food:  b2b_reg_search_IT_VER_Foo_ +numeric value (max link 240 characters)
 * _UK__MAN_Bea_0000(7 numeric  digits)
